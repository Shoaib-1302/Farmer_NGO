# Greenipath Assignment — Makkala Vikasa (NGO_1) & YIDS (NGO_2) Plantation Data

Data cleaning, GIS validation, overlap detection, and an interactive dashboard for two
NGOs' farmer plantation field data (Kobo Collect Excel exports + KML/KMZ boundary files).

## How to run

### 1. Install dependencies
```bash
pip install pandas numpy openpyxl shapely pyproj folium streamlit plotly streamlit-folium
```

### 2. Run the pipeline (in order — each step reads the previous step's output)
```bash
cd 3_Scripts
python3 data_cleaning.py       # -> 2_Cleaned_Data/Duplicate_Records.xlsx, Data_Quality_Report.xlsx
python3 kml_matching.py        # -> 2_Cleaned_Data/GIS_Validation_Report.xlsx
python3 overlap_analysis.py    # -> 2_Cleaned_Data/Cleaned_Master.xlsx
```
Each script also writes a small interim cache to a `_interim/` folder at the project
root (pickled DataFrames) so the next script doesn't have to redo the previous step's
work. That folder is safe to delete — it's regenerated automatically.

### 3. Run the dashboard
```bash
cd 3_Scripts        # or: cd 5_Dashboard
streamlit run dashboard_app.py     # or: streamlit run app.py
```
The dashboard reads `2_Cleaned_Data/Cleaned_Master.xlsx` directly, so it only needs the
three pipeline scripts to have been run once — after that you can re-run the dashboard
on its own any time. 8 tabs: Overview (reliability score, before/after impact),
Data Quality (issue-category breakdown), GIS Validation (area scatter, suspicious-GPS
clusters), Species & Carbon (with confidence labels + assumptions), Map, Villages
(top problematic villages), Farmer Table, and Priority Actions.

The Map tab uses a clustered, zoomable Folium map embedded via `streamlit-folium`, with
one cluster layer per severity tier (🟢 Clean / 🟡 Warning / 🔴 Critical) plus a
dedicated marker style for suspicious duplicate-GPS points — markers merge into
clusters when zoomed out and separate into individual plots as you zoom in. A checkbox
toggles a "Polygon Overlaps" overlay showing every overlapping-plot pair as a connecting
line, colored by severity (red ≥20% overlap, amber 10–20%). Everything lives in the
dashboard — no separate map HTML files are generated.

## About `6_Report/Final_Report.pdf`
This is a generated deliverable (a narrative write-up of findings for a non-technical
audience) and, like the other generated outputs, isn't pre-built into this package —
`Methodology.md` in the same folder already contains the full methodology and headline
numbers; a PDF can be produced from it (e.g. via `pandoc Methodology.md -o Final_Report.pdf`)
once you've reviewed the results from your own run.

## Key thresholds (adjustable at the top of the relevant script)
- Phone format: 10-digit Indian mobile number starting 6–9
- Name-clustering similarity: 0.84 (`difflib.SequenceMatcher` ratio)
- GPS-vs-KML-centroid mismatch: > 300 m
- Claimed-vs-KML area mismatch: > 15%
- Plot overlap flag: ≥ 10% of either plot's area (Critical tier at ≥ 20%)
- Valid GPS region: lat 10–20°N, lon 72–80°E (project area)
- Suspicious duplicate-GPS cluster: ≥3 records from ≥2 different farmers at the same
  ~1m location, inside the valid region

See `6_Report/Methodology.md` for the full write-up, including headline results from the
sample data provided.
